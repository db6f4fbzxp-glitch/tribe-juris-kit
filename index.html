The Architecture

The Four-Brain System

Your finished engine is a four-brain stack: three server-side minds running on your Mac plus you as the commander. Each brain plays a distinct role. Together they form a complete legal research and drafting environment — sovereign, integrated, and tuned to you.

The Stack Components

  • YOU — The commander. You give orders, set strategy, hold final authority on every decision. The other three brains exist to serve and protect what you're building.
  • COWORK (Claude desktop app on your Mac) — Your drafter, executor, strategy partner. Builds documents, queries MEMORY and JURIS on your behalf, ties everything together. Has multiple arms:
    • — Native arm: file system, native macOS UI control, AppleScript, Terminal
    • — Chrome arm: Claude in Chrome MCP — browser automation for JEDS dockets, county recorders, government databases, GitHub, anything web
  • JURIS (AnythingLLM, port 3001) — Your legal brain. Holds your library of authoritative documents, returns retrieval-grounded answers with citations. Uses your personalized system prompt as the steering wheel.
  • MEMORY (AgentMemory v0.9.21, port 3111) — Persistent memory across all sessions. Decisions, locked arguments, citation rules, work continuity. LLM via Claude API; embeddings via Ollama nomic-embed-text routed through the OpenAI-compatible /v1 endpoint.
Chrome arm — install the extension Claude in Chrome is the browser arm of COWORK. Install the official Chrome extension from claude.ai/chrome, sign in with your same Claude account, and your Cowork sessions can now drive any browser tab — pull court dockets from JEDS, scrape county recorders, fill forms, navigate government sites, screenshot evidence. Step 06 (MCP Bridges) ensures Claude in Chrome talks to your AgentMemory too, so browser-session work shares memory with the rest of your stack.

Hard-Learned Law

The Eight Sacred Rules

Codified from Operation Van Zile's Tiddy Mini M4 session (May 23, 2026) — a six-hour debugging marathon that earned the right to be law. Honor them. Every script in this kit follows them. If you write your own scripts, follow them too.

AgentMemory v0.9.x needs Ollama via /v1 — not the native API endpoint

The doctor command in v0.9.16 reports a false negative on native Ollama embedding detection. Community fix (GitHub Issue #232): route Ollama through its OpenAI-compatible /v1 endpoint. Your ~/.agentmemory/.env must include EMBEDDING_PROVIDER=openai, OPENAI_BASE_URL=http://127.0.0.1:11434/v1, OPENAI_API_KEY=ollama.

v0.9.21+ requires a real Anthropic API key for the LLM side

Older versions tolerated Ollama as LLM provider. v0.9.21 is stricter — set ANTHROPIC_API_KEY=sk-ant-... in .env or compression, graph extraction, and consolidation all silently die.

macOS ships BSD versions of Unix tools — never use GNU-only flags in your scripts

rsync --info=progress2 → use --progress. du -b → use du -k (multiply by 1024 for bytes). stat -c %s → use stat -f %z. find -printf → use -exec or -print. To get GNU tools: brew install coreutils rsync, then prefix with g (gdu, grsync).

Never use rsync -a or cp -R on macOS bundle files

.photoslibrary, .musiclibrary, .imovielibrary, .keynote, .app bundles contain SQLite databases, resource forks, hardlinks, and extended attributes that rsync -a mangles. Symptom: copy completes successfully, file counts match, but the app sees an empty library. Use ditto "$SRC" "$DEST" instead — Apple's purpose-built bundle copier preserves everything.

Photos Library cannot relocate to HFS+ external when iCloud Photos is enabled

Modern macOS Photos refuses to function as System Photo Library from an HFS+ external drive. Workaround: keep Photos on internal, OR reformat external to APFS (data-destroying — only viable on empty drives).

Snapshot the binary store before every reboot

AgentMemory's binary store at ~/data/state_store.db/mem%3Amemories.bin is the truth. The launcher in Step 07 includes pre-start backup rotation that keeps the 5 most recent copies. Don't disable it. Memories saved late in a session can be lost on reboot if no snapshot occurred in between.

Discovery-first principle

Before deleting or moving anything: scan first, identify, flag sacred zones, then act. Every script in this kit does this. If you write your own, follow the pattern. Read-only scans cost almost nothing; recovery from a wrong delete can cost everything.

The Do-Not-Touch List

Once your stack is built, these paths are sacred: ~/Library/Application Support/anythingllm-desktop/ (your JURIS vector DB), ~/Library/Application Support/Claude/ (Cowork data), ~/.agentmemory/ (AgentMemory configuration and data), ~/data/state_store.db/ (binary store), your RAZOR_INTELLIGENCE folder, your Ollama models directory.

Read Twice

API Key Hygiene

⚠ Sacred
  1. Never paste your sk-ant-... key in a chat, screenshot, screen-share, or any place that gets logged. Not to other tribe members. Not to Cowork. Not to anyone.
  2. Never commit it to any git repository, public or private.
  3. Keep it only in .env files on your machine and in a password manager.
  4. If exposed — even briefly — revoke at console.anthropic.com immediately and create a new one. It is free to create new keys.
  5. Every script in this kit uses interactive hidden input (read -s pattern) so the key never appears on screen or enters shell history. Use the scripts. Don't paste keys into one-off commands.
